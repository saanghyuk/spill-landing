# Spill — 서비스 기획서

## 1. 서비스 개요

**서비스명**: Spill
**슬로건**: Real you. Real me.
**한 줄 정의**: 후면카메라 SNS — AI가 매일 질문을 던지면, 사용자가 후면카메라로 자기 세상을 비추면서 목소리로 답하는 SNS
**컨셉 키워드**: "후면카메라 SNS" — 전면카메라는 네 얼굴을 보여주고, 후면카메라는 네 세상을 보여준다.
**핵심 원칙**: AI는 질문만 한다. 콘텐츠는 100% 사용자가 만든다. 편집 없음. 필터 없음. 대본 없음. 후면카메라 기본.

**핵심 톤**: "뭘 말해야 할지 모르겠다고? 우리가 물어볼게. 넌 답만 해."
- "꾸미기 그만해" 같은 공격적 톤이 아님
- "편하게 답해, 그게 전부야" 같은 초대하는 톤
- "뭘 올리지?"라는 SNS 고유의 부담을 질문으로 제거

**타겟 유저**: 꾸며진 SNS에 피로감을 느끼지만 표현 욕구는 있는 20대 중후반. 할 말은 있는데 계기가 없었던 사람들.

## 2. 질문 시스템

매일 사용자에게 **2개의 질문**이 제공된다. 두 질문은 "메인/보조"가 아니라 **동등한 두 축**.

> **"하나는 모두에게. 하나는 나에게."**

### 2-1. 공통 질문 — 광장 (1개/일)

- **도착 시간**: 오전 9시 KST
- **만료**: 12시간 후 (오후 9시)
- 모든 유저에게 동일한 질문
- 역할: 모르는 사람끼리 같은 주제로 연결되는 **광장**
- 동기: "다른 사람은 뭐라고 답했지?" → 피드 소비 유도
- 오늘의 공통 질문은 피드 상단에 고정 노출

**질문 예시**:
- "지금 솔직히 말 못 하고 있는 게 뭐예요?"
- "새벽 2시에 자꾸 생각나는 게 뭐예요?"
- "보고 싶은데 먼저 연락 안 하는 사람 있어요?"
- "자기한테 제일 자주 하는 거짓말이 뭐예요?"
- "미루고 있는 거, 사실 뭔지 알잖아요?"
- "샤워하면서 혼자 무슨 대화해요?"

### 2-2. 개인화 질문 — 일기장 (1개/일)

- **도착 시간**: 오후 9시 KST
- **만료**: 12시간 후 (오전 9시)
- 사용자별 다른 질문 — AI가 과거 답변 분석 후 생성
- 역할: 나만의 이야기를 이어가는 **일기장**
- 동기: "AI가 나한테 뭐라고 물어봤지?" → 앱 재진입 유도
- 꼬리를 무는 질문: 친구가 자연스럽게 이어서 물어보는 느낌
- 처음 가입한 유저: 온보딩 음성 답변 기반으로 첫 개인화 질문 생성
- 답변 0개인 유저: 공통 질문만 제공, 답변 3개 쌓이면 개인화 시작
- 답변은 모두 공개 (공통 질문과 동일)

**개인화 질문 예시** (이전 답변 기반):
- 유저가 "회사 그만두고 싶어요" 라고 답함 → **"그만두면 제일 먼저 뭐 할 거예요?"**
- 유저가 "오랜만에 엄마랑 통화했어요" → **"엄마한테 아직 못 한 말 있어요?"**
- 유저가 "요즘 혼자가 편해요" → **"그게 평화인 거예요, 아니면 뭔가 피하는 거예요?"**

**왜 두 질문이 동등한가**:
- 공통 질문만 있으면 → "모두를 위한 앱" (개인적이지 않음)
- 개인 질문만 있으면 → "혼자 쓰는 일기" (소셜이 아님)
- 둘 다 있으니까 → **광장에서 사람 만나고 + 집에서 나를 돌아보는** 완전한 루프

### 2-3. 질문 톤 가이드라인 (AI 프롬프트 반영)

- 친한 친구가 편하게 묻는 말투
- 존댓말이지만 딱딱하지 않게
- **질문 깊이 비율: 가벼운 60% + 중간 20% + 깊은 20%**
  - 가벼운: "샤워하면서 뭐 해요?" "요즘 제일 많이 듣는 노래?"
  - 중간: "요즘 자꾸 미루는 거 하나만?" "솔직히 지금 직업 만족해요?"
  - 깊은: "부모님 닮기 싫은 부분?" "마지막으로 진심으로 운 게 언제?"
  - **가벼운 질문이 60% 이상이어야 불행 경쟁을 방지**
- **절대 금지**: 설문지 느낌, 심리테스트 느낌, 인터뷰 느낌, 취조 느낌
- 질문을 읽었을 때 "아 나 이거 대답하고 싶다" 가 나와야 함

### 2-4. 질문 만료 규칙

- 모든 질문은 **12시간 후 사라짐**
- 공통 질문: 오전 9시 도착 → 오후 9시 만료
- 개인 질문: 오후 9시 도착 → 오전 9시 만료
- 하루에 두 번 앱을 열 이유가 생김 (아침 + 저녁)
- 답변 안 한 질문은 그냥 사라짐 (알림/압박 없음)
- 답변한 질문 + 답변 영상은 내 아카이브에 영구 저장
- **핵심 카피**: "모든 질문은 12시간 뒤에 사라져요. 하지만 당신의 답은 영원히 남아요."

## 3. 온보딩

**목적**: 개인화 질문 생성을 위한 최소한의 정보 수집

**플로우**:
1. 소셜 로그인 (카카오 / 애플 / 구글)
2. **음성 온보딩 = 나의 첫 번째 Spill**: 공통 질문 형태로 가볍게 던짐 → 녹화 버튼 → 영상 답변
   - **부담 안 주는 게 핵심** — "당신에 대해 말해보세요" 같은 막막한 질문 금지
   - 온보딩 전용 질문 풀 (가볍고 답하기 쉬운 것들):
     - "요즘 제일 많이 듣는 노래가 뭐예요?"
     - "오늘 점심 뭐 먹었어요?"
     - "지금 기분을 한 단어로 하면?"
     - "어제 가장 웃긴 일이 뭐였어요?"
   - 최대 1분, 최소 없음
   - 이게 Spill의 첫 경험 — 온보딩 자체가 서비스 체험
   - 녹화 영상은 STT 변환 → AI 분석 → 첫 개인화 질문 생성
   - 이 영상도 공개됨 (모든 답변은 공개 원칙)
3. 스킵 가능 → 공통 질문만 제공, 답변 3개 후 개인화 시작

**왜 음성 온보딩인가**:
- 텍스트 입력은 Spill의 "말로 하는" 철학과 안 맞음
- 첫 경험부터 "녹화 → 말하기"를 체험시켜야 두 번째가 쉬움
- 텍스트보다 음성이 더 풍부한 정보 → 개인화 질문 품질 향상
- "아 이런 식으로 하는 거구나"를 설명이 아닌 행동으로 알려줌
- **부담 없는 질문이 핵심**: 첫 경험이 무거우면 이탈. "오늘 점심 뭐 먹었어요?"면 누구나 답할 수 있음

**팔로우 추천** (온보딩 마지막 단계):

단순 나열이 아니라 **"내 피드를 재밌게 만들어줄 사람"** 으로 의미 있게 구성.
목표: **공감(Empathy) + 시야 확장(Broaden Perspective)**

- **"나와 비슷한 사람"** 섹션
  - 온보딩 답변 분석 → 비슷한 키워드/감정으로 답한 유저
  - "이 사람도 비슷한 이야기를 했어요"
  - 목적: 공감 — "나만 이런 게 아니구나"

- **"나와 다른 시각"** 섹션
  - 같은 질문에 다른 관점으로 답한 유저
  - "이 사람은 완전 다르게 생각했어요"
  - 목적: 시야 확장 — 내가 몰랐던 관점

- **"요즘 공명이 큰 사람"** 섹션
  - 최근 답변에 공명이 많이 발생한 유저
  - "많은 사람이 이 사람 이야기에 공명했어요"
  - 목적: 퀄리티 — 재밌는 사람을 바로 만남

- **"내 주변 사람"** 섹션
  - 연락처 / 이메일 / 소셜 로그인 친구 연동
  - "이미 아는 사람의 솔직한 모습, 궁금하지 않아요?"
  - 목적: 아는 사람의 새로운 면 발견

- 온보딩에서 최소 3명 팔로우 권장 (강제 아님)
- 설정 내 "친구 찾기"에서 언제든 재접근 가능

## 4. 답변 (콘텐츠 업로드)

### 4-1. 형식 규칙

- **영상 + 목소리 필수** — 텍스트 전용 답변 불가
- **카메라 기본값 = 후면카메라** (내가 보고 있는 것 = 내 시선)
  - 전면카메라 전환 가능하되, 한 번 더 탭 필요 (넛지)
  - 강제가 아닌 유도 → 자연스럽게 대부분의 영상이 POV가 됨
  - 컨셉: "네 얼굴이 아니라 네가 보는 세상을 보여줘"
- 최소 길이 없음
- **최대 길이**: 1분 (60초)
  - "1분만 솔직하면 돼" — 부담 극소, 밀도 극대
  - 보는 사람도 부담 없음 → 스와이프 리텐션 상승
  - 짧으니까 "하나 더 할까?" 가 됨
- **편집 기능 없음**: 필터, 자막, 트리밍 전부 없음
- 녹화 → 미리보기 → 다시 찍기 or 올리기 (올린 후에는 재녹화 불가)
- 같은 질문에 답변 1개만 가능
- **모든 답변은 공개** — 공개/비공개 선택지 없음
  - 어차피 닉네임 + 후면카메라(얼굴 안 나옴) + 목소리만 = 충분히 익명
  - "여기선 다 공개야. 근데 걱정 마. 아무도 네가 누군지 몰라."
  - 비공개 옵션이 있으면 대부분 숨음 → 콘텐츠 부족 → 서비스 사망
  - 비공개 없으니까 선택 피로 없음, 업로드 플로우 단순해짐
- 답변 삭제 가능 (삭제 후 같은 질문에 재답변 가능, 12시간 유효 기간 내에만)
- 삭제된 답변 복구 불가
- **삭제가 유일한 안전망** — "후회하면 지울 수 있다"는 심리적 안전감

### 4-2. 업로드 플로우

1. 질문 카드에서 "답하기" 탭
2. **후면카메라로 열림** (내 시선 = 기본)
3. 녹화 시작 → 타이머 60초 카운트다운
4. 자동 종료 or 수동 종료
5. 미리보기 → 다시 찍기 or 올리기
6. 업로드 완료 → 피드로 이동

- 공개 범위 설정 단계 없음 → **녹화 → 확인 → 올리기** 3스텝으로 단순화

### 4-3. 철학

**"네가 보는 세상을 보여주고, 네 목소리를 들려줘. 1분이면 돼."**

유저가 걱정할 필요 없는 것들:
- 뭘 말할지 (질문이 알아서 해줌)
- 어떻게 보이는지 (얼굴 안 나옴, 내 시선만 보임)
- 재미있어야 하는지 (솔직하기만 하면 됨)
- 오래 말해야 하는지 (1분이면 충분)
- 잘해야 하는지 (잘할 필요 없음, 그냥 너만 되면 됨)

**핵심 카피**: "멋있고 예쁠 필요 없어요. 솔직하기만 하면 돼요."

## 5. 피드

### 5-1. 피드 구조

질문 중심 피드 (인스타그램처럼 시간순 팔로우 피드가 아님)

### 5-2. 피드 탭 구성

**탭 1 — 오늘 (기본 탭)**
- 상단: 오늘의 공통 질문 카드 (고정)
- 아래: 공통 질문에 답한 사람들의 영상 목록
- **답변 게이트**: 답하기 전에는 다른 사람 답변 3개만 보임 (나머지 블러). 답변 후 전체 피드 열림
- 알고리즘 (3단계):
  - 1순위 (40%): 팔로우한 친구들의 답변
  - 2순위 (30%): 비슷한 성향의 모르는 유저 (과거 공명 패턴 기반)
  - 3순위 (30%): 다른 시각의 모르는 유저 (같은 질문에 다른 키워드로 답한 사람)
  - 신규 유저 (팔로우 없음): 비슷한 성향 50% + 다른 시각 50%

**탭 2 — 나의 질문**
- 오늘 나에게 온 개인화 질문
- 미답변: 답변 유도 UI
- 답변 완료: 내 답변 + 비슷한 개인화 질문을 받은 다른 유저 답변 (있는 경우)

**탭 3 — 아카이브**
- 내가 과거에 답한 모든 영상 목록
- 날짜별 정렬
- **"1년 전 오늘"** 섹션 상단 고정 (해당 날짜 답변 있을 경우)
- 답변 안 한 날은 "skipped"으로 표시 (패널티 없음, 그냥 공백)

### 5-3. 영상 재생 방식

- 쇼츠/릴스 형태: 세로 전체화면, 위아래 스와이프로 다음 답변
- 피드 진입 시 첫 영상 자동재생 (무음 시작 → 탭하면 소리 켜짐)
- 스와이프하면 다음 영상 자동재생

### 5-4. 사람 발견 메커니즘

**"답변할수록 더 많은 사람을 만나요 — 나와 비슷한 사람, 그리고 나와 완전히 다른 사람."**

- 답변 많이 할수록 → AI가 프로필 더 잘 이해 → 더 좋은 사람 매칭
- 비슷한 사람: 같은 주제에 공명한 유저
- 다른 사람: 같은 질문에 다른 관점으로 답한 유저
- 이것이 질문 자체를 넘어서는 핵심 리텐션 루프

## 6. 반응 시스템

**댓글 없음.** 3가지 반응만 존재.

### 6-1. 공명 (Resonate)

- 탭 → 프리셋 중 선택:
  - **가벼운 반응** (가벼운 질문에도 자연스럽게):
    - "ㅋㅋㅋ 완전 공감"
    - "와 나도!"
    - "이거 맞아요"
  - **감정 반응** (깊은 질문에 어울리는):
    - "듣고 싶었어요"
    - "용기가 돼요"
    - "안아주고 싶어요"
- 프리셋 선택 안 하면 기본 공명
- 공명 수는 숫자가 아닌 **물결(파장) 크기**로 시각화
- 숫자는 탭해야 보임
- **왜 가벼운 반응이 필요한가**: 감정적 반응만 있으면 무거운 답변만 보상받는 구조 → 불행 경쟁 유발. 가벼운 반응이 있어야 "샤워하면서 노래한다"는 답변에도 자연스럽게 공명 가능
- 공명한 사람 목록 조회 가능 (프로필 + 선택한 프리셋)

### 6-2. 서로 공명 (Mutual Resonance)

- A가 B에게 공명 + B가 A에게 공명 → "서로 공명했어요" 알림
- 서로 공명한 사람끼리는 귓속말(DM)이 **수락 없이 바로 열림** *<u>(이건 소개팅 앱이 이렇게 한다고 함ㅋㅋ)</u>*
- Spill의 자연스러운 연결 메커니즘

### 6-3. 나도 답하기 (Answer This Too)

- **핵심 바이럴 메커니즘**
- 탭 → 해당 질문의 답변 녹화 화면으로 바로 이동
- 이미 답한 경우: "이미 답했어요. 다시 볼까요?"
- 심리: 남의 솔직함을 보면 나도 솔직해지고 싶어짐
- **핵심 카피**: "누군가의 답변을 듣다 보면 나도 답하고 싶어져요."

### 6-4. 귓속말 (Whisper) — 잠금 해제형 옵션 기능

귓속말은 처음부터 열려있지 않다. **일정 활동 이후 해제되는 보상형 기능.**

**해제 조건**: 답변 **5개 이상** 올린 유저에게만 활성화
- 5개 미만: 귓속말 버튼 보이되, 탭 시 "답변 5개를 올리면 귓속말이 열려요" 안내
- 5개 이상: 기능 해제 + 축하 알림 "귓속말이 열렸어요"

**왜 잠금 해제형인가**:
- **자연스러운 DM 스크리닝**: 답변 5개 = 이미 자기 목소리/환경/생각을 공개한 사람. 익명 뒤에 숨어서 이상한 DM 보낼 수 없음. "나"를 이미 밝힌 상태이기 때문에 개같은 DM이 자연스럽게 걸러짐.
- 처음부터 DM 열리면 → 스팸/훈수/헌팅 리스크
- "답변 더 올리면 새 기능 열림" → 추가 리텐션 동기
- 귓속말이 "열심히 한 사람만의 것" → 프리미엄 느낌

**규칙**:
- 1:1 비공개 DM, 텍스트 전용 (영상/이미지 불가)
- 팔로우 관계 아닌 경우: 상대방 수락 후 대화 시작
- 예외: 서로 공명한 사이는 수락 없이 바로 대화 가능
- 공개 공간은 깨끗하게 유지, 깊은 대화는 귓속말로

## 7. 알림 시스템

### 있는 알림:
- 오늘의 질문 도착 (공통: 오전 9시, 개인: 오후 9시)
- 누군가 내 답변에 공명함 (프리셋 메시지 포함)
- 누군가 "나도 답하기"를 눌러 답변 올림
- 귓속말 수신 (5개 이상 답변 유저만)
- "1년 전 오늘 당신의 답변이 있어요" (해당 날짜 있을 경우)

### 없는 알림:
- 팔로워 수 변화
- 답변 조회수
- 연속 답변 스트릭 관련 어떤 알림도 없음
- "오늘 아직 답 안 했어요" 같은 재촉

**철학**: 죄책감 없음. 압박 없음. 허세 지표로 사용자를 끌어오지 않음.

## 8. 심리적 설계 — 솔직함을 만드는 구조

### 8-1. 핵심 원칙: "잘 보이고 싶은 본능"을 적이 아닌 아군으로

사람은 본능적으로 잘 보이고 싶다. 이걸 없애는 게 아니라 **"뭐가 잘인지"의 기준을 바꾸는 것.**

| 인스타에서 "잘" | Spill에서 "잘" |
|---|---|
| 예쁘게 | 솔직하게 |
| 성공한 것처럼 | 진짜인 것처럼 |
| 부러워 보이게 | 공감 가게 |

"솔직한 게 쿨하다"는 문화를 **강제하지 않고 구조로 만든다:**
- 편집 불가 → 꾸밀 수가 없음
- 질문이 구체적 → 있어보이기 어려움 ("샤워하면서 뭐 해요?"에 폼 잡기 어려움)
- 꾸민 답변은 공명이 안 옴 → "아 솔직하게 해야 반응 오는구나" 학습
- 솔직한 답변에 공명 폭발 → **솔직함 = 사회적 보상** 구조 형성

### 8-2. Beautiful Mess Effect 활용

심리학 연구 결과: 사람들은 **자기 자신의 솔직함은 약점**으로 느끼지만, **남의 솔직함은 용기**로 느낀다.

**Spill에서 이걸 활용하는 방법:**
- 답변 올리기 전: "솔직한 답변일수록 공명이 많아요" 넛지
- 첫 공명 받았을 때: 특별한 경험 ("누군가에게 당신의 이야기가 닿았어요")
- 다른 사람 답변 보여주기: "이 사람도 처음엔 긴장했어요" (정규화)
- 핵심: **"나만 부끄럽다고 느끼지, 보는 사람은 멋있다고 느낀다"**를 체험시키기

### 8-3. Intimacy Ramp (점진적 깊이)

첫날부터 "제일 무서운 게 뭐예요?" 하면 이탈. **자연스러운 관계처럼 점진적으로.**

| 유저 답변 수 | 질문 깊이 | 예시 |
|---|---|---|
| 0~3개 | 가벼운 것만 | "요즘 제일 많이 듣는 노래?" |
| 4~10개 | 가벼운 70% + 중간 30% | "솔직히 지금 직업 만족해요?" |
| 11~20개 | 가벼운 50% + 중간 30% + 깊은 20% | "부모님한테 못 한 말 있어요?" |
| 20개 이상 | 전체 밸런스 (60/20/20) | 모든 패턴 오픈 |

- 이건 **개인화 질문에만 적용** (공통 질문은 모두에게 같으니까)
- 유저가 모르게 자연스럽게 깊어짐
- 심리학의 Social Penetration Theory(사회적 침투 이론) 기반: 관계는 양파 껍질처럼 점진적으로 깊어짐

### 8-4. 불행 경쟁 방지

**위험**: "솔직함이 쿨하다" → "더 솔직해야 더 쿨하다" → "더 불행한 이야기 = 더 많은 공명" → **불행 경쟁**

**방지 장치:**
- 질문 비율: **가벼운 60% 이상 유지** — 매일 무거운 질문 나오면 피곤
- 공명 프리셋에 가벼운 반응 ("ㅋㅋㅋ 완전 공감") → 재밌는 답변도 보상받음
- 시즈널 질문: 월요일, 금요일, 날씨 등 가벼운 것 활용
- 공명 수 시각화를 **숫자가 아닌 물결**로 → 정량 비교 불가 → "누가 더 많이 받았나" 경쟁 방지
- 패턴5 "지금 이 순간" 질문 활용: 깊이 없이도 재밌는 콘텐츠

### 8-5. 심리적 안전감을 만드는 구조

| 요소 | 설계 | 효과 |
|---|---|---|
| **가명(Pseudonymity)** | 닉네임 사용, 실명 아님. 단 목소리는 나옴 | 완전 익명의 독성 방지 + 실명의 부담 제거. 적절한 책임감 |
| **후면카메라 기본** | 얼굴 안 나옴. 내 시선만 보임 | 외모 판단 없음 → 안전감 |
| **전부 공개, 하지만 익명** | 공개/비공개 선택 없음. 대신 닉네임+후면카메라로 충분히 익명 | 선택 피로 제거 + 콘텐츠 풍부 + "어차피 모르니까" 솔직함 촉진 |
| **삭제만 가능** | 비공개는 못 하지만 후회하면 지울 수 있음 | 유일한 안전망 → "최악의 경우 지우면 돼" → 일단 올리게 됨 |
| **댓글 없음** | 공명 프리셋만 | 비판/훈수/트롤링 구조적 차단 |
| **공명은 긍정만** | 프리셋이 전부 긍정적 | "이상하게 생각하진 않을까?" 걱정 제거 |
| **"나도 그래요" 보여주기** | 비슷한 답변 한 다른 유저 매칭 | "나만 이런 게 아니구나" → 소속감 |
| **작은 그룹 느낌** | "2,847명이 답했어요" 대신 "당신과 비슷한 47명이 답했어요" | 광장이 아닌 소모임 느낌 → 안전감 |

### 8-6. 첫 공명의 마법

유저가 첫 답변 올리고 첫 공명을 받는 순간이 **서비스의 생사를 가른다.**

- 아무 공명 없음 → "나만 이런가" → 이탈
- 첫 공명 받음 → "아 누가 들어줬네" → 습관 형성

**첫 공명을 보장하는 장치:**
- 신규 유저 답변은 피드에서 우선 노출 (24시간 한정)
- "이 사람의 첫 번째 Spill이에요" 배지 → 기존 유저가 공명해주고 싶어짐
- 콜드스타트 기간: 운영팀/시드 유저가 신규 유저 답변에 적극 공명

### 8-7. 콜드스타트 대응

**피드가 비어있으면 아무리 컨셉이 좋아도 의미 없다.**

| 상황 | 대응 |
|---|---|
| 피드에 답변 0개 | "아직 아무도 답하지 않았어요. 첫 번째가 되어보세요." + 시드 유저 답변 즉시 투입 |
| 공명 0개 (올린 지 1시간) | 시드 유저/운영팀이 공명 (초기에만) |
| 팔로잉 0명 | 팔로우 추천 강화 + "이 사람들의 답변을 먼저 들어보세요" |
| 개인 질문 아직 없음 (답변 3개 미만) | "답변 3개를 올리면 AI가 당신만의 질문을 만들어드려요" 진행 바 표시 |

### 8-8. 시드 유저 전략 — 첫 100명이 문화를 만든다

**시드 유저 선정 기준:**
- 인플루언서/크리에이터가 아닌 **"솔직한 일반인"**
- 마이크로 인플루언서 중 "날것" 콘텐츠 하는 사람
- 20대 중후반, 다양한 직업/배경
- 핵심: 이 사람들의 답변을 보고 **"아 이 앱에서는 이렇게 하면 되는구나"** 가 형성됨

**시드 유저 가이드:**
- "예쁘게 찍으려 하지 마세요. 지금 보이는 것 그대로 비추세요."
- "생각나는 대로 말하세요. 정리 안 해도 돼요."
- "어색해도 괜찮아요. 어색한 게 진짜예요."
- 첫 1주일: 운영팀이 시드 유저들의 답변에 적극 공명 → 보상 구조 학습시킴

**시드 유저 답변 예시 (이런 느낌이어야 함):**
- 천장 비추면서 "솔직히 요즘 좀 외로운데... 말할 데가 없었어요"
- 걷는 바닥 비추면서 "퇴근길인데 아까 팀장이 한 말이 자꾸 생각나요"
- 어두운 방에서 "지금 3시인데 잠이 안 와서 그냥 켰어요"

### 8-9. 감성 장치 — 숫자가 아닌 서사로 보여주기

Spill의 모든 피드백은 **"숫자"가 아니라 "이야기"**로 전달한다.

**A. 기록의 감성**

| 순간 | 메시지 | 효과 |
|---|---|---|
| 답변 올린 직후 | "오늘의 당신이 기록됐어요." | 업로드 완료가 아닌 "기록"의 소중함 |
| 답변 3개 | "당신에 대해 3가지를 알게 됐어요." | 자기 발견 시작 |
| 답변 10개 | "10번의 솔직함. 당신의 이야기가 쌓이고 있어요." | 여정의 느낌 |
| 답변 30개 | "한 달간의 당신. 이게 진짜 당신이에요." | 정체성 |
| 답변 100개 | "100번의 솔직함. 이게 당신의 이야기예요." | 뿌듯함 + 이탈 방지 |

**B. 연결의 감성**

| 순간 | 메시지 | 효과 |
|---|---|---|
| 첫 공명 받음 | "누군가에게 당신의 이야기가 닿았어요." | 혼자가 아닌 느낌 |
| 공명 알림 | "당신의 이야기가 3명에게 닿았어요." | "공명 3개"가 아닌 "닿았어요" |
| 서로 공명 | "서로의 이야기에 닿았어요." | 연결의 따뜻함 |
| 같은 사람 답변 3개 봄 | "이 사람의 이야기를 3번 들었어요." | 관계 형성 |
| 누군가 나도 답하기 | "당신의 답변이 누군가를 움직였어요." | 영향력의 느낌 |

**C. 시간이 만드는 감성**

| 순간 | 메시지 | 효과 |
|---|---|---|
| 주간 리캡 (일요일) | "이번 주 당신은 4번 솔직했어요." (압박 아닌 리캡) | 뿌듯함 |
| 월간 하이라이트 | "지난 달 가장 많은 사람에게 닿은 이야기" | 자기 성찰 |
| 1년 전 오늘 | "1년 전 오늘의 당신. 들어볼래요?" | 타임캡슐 |
| 연말 리캡 | "올해의 Spill — 당신이 가장 많이 이야기한 것: 가족, 일, 꿈" | Spotify Wrapped식 자기 발견 + SNS 공유 바이럴 |

**D. 핵심 원칙: 숫자 → 서사**

| 기존 SNS | Spill |
|---|---|
| "좋아요 47개" | "47명에게 닿았어요" |
| "팔로워 230명" | "230명이 당신의 이야기를 듣고 있어요" |
| "조회수 1,200" | 조회수 표시 없음 |
| "연속 7일 달성!" | "이번 주 당신은 5번 솔직했어요" (스트릭 아닌 리캡) |
| "프로필 완성도 80%" | "당신에 대해 10가지를 알게 됐어요" |

> **"혼자 쓰는 일기는 오래 못 가요. 근데 누군가 읽어주는 일기는 계속 쓰고 싶어져요."**
> 이게 Spill의 본질. 기록이면서 소통. 나를 알아가면서 너를 알아가는 시간.

## 10. 타임캡슐 (아카이브)

모든 답변은 영구 기록이 된다.

- 유저의 개인 아카이브에 저장
- 날짜별 정렬
- **"1년 전 오늘"**: 현재 날짜와 정확히 1년 전 답변이 일치하면 알림 + 아카이브 탭 상단에 표시
- **핵심 카피**: "1년 뒤, 오늘의 당신 목소리를 다시 듣게 될 거예요."
- 답변 안 한 날은 "skipped"으로 표시 (벌점 없음, 그냥 빈 칸)
- **"매일 답하지 않아도 괜찮아요. 하지만 답하면, 그건 영원히 남아요."**

## 11. 성장 & 바이럴 & 심리 전략

### 11-1. 초대제 시스템

**가입은 초대장이 있어야만 가능하다.**

- 모든 신규 유저에게 **초대장 3장** 지급
- 초대장은 고유 코드 또는 링크 형태
- 초대받은 사람이 가입 + 첫 답변 완료 시 → 초대한 사람에게 **추가 초대장 1장** 보상
- 초대장 잔여 수는 프로필에서 확인 가능
- 초대한 사람 목록 조회 가능 ("내가 초대한 사람들")

**왜 3장인가**:
- 2장(클럽하우스)은 너무 적어서 "아까워서 안 줌" 발생
- 5장 이상은 희소성이 희석됨
- 3장 = "진짜 이 사람한테 주고 싶다" 고민하게 만드는 적정 수
- 친구가 답변하면 1장 더 받으니까 → 초대만 하는 게 아니라 **친구가 실제로 쓰게 만드는** 동기 부여

**초대장 바이럴 시나리오**:
1. A가 Spill 시작, 초대장 3장 받음
2. A가 친구 B, C, D에게 초대
3. B가 가입 + 첫 답변 → A에게 초대장 +1
4. B의 답변을 A가 봄 → A가 "나도 답하기" → 앱 내 활동 증가
5. B도 초대장 3장 받음 → B의 친구 E, F, G에게 초대
6. 지수적 성장

**초대장 FOMO 설계**:
- 랜딩 페이지: "초대장이 있어야 들어올 수 있어요"
- SNS에서 자연스럽게: "나 Spill 초대장 있는 사람?" 이 바이럴 됨
- 초대장 없이 사전등록 → 대기열에 등록 → "앞에 847명 대기 중" 표시

### 11-2. 런칭 전 바이럴 전략

**Phase 1 — 시드 유저 확보 (사전등록)**
- 랜딩 페이지 오픈, 이메일 수집
- "First 1,000" 희소성
- 질문 로테이션으로 서비스 체험 느낌 제공
- 대기열 순번 표시 → 공유하면 순번 올라가는 메커니즘 (선택)

**Phase 2 — 시딩 (베타 100명)**
- 마이크로 인플루언서 10~20명에게 먼저 초대
- 크리에이터가 아니라 **솔직한 사람** 위주 (일반인 중 표현력 있는 사람)
- 이 사람들의 답변 영상 클립을 티저로 SNS에 공개
- "이 사람이 Spill에서 이런 답변을 했다" → 궁금증 유발

**Phase 3 — 초대제 오픈 (1,000명)**
- 시드 유저 100명에게 초대장 3장씩 = 300명 추가 가능
- 300명의 초대장 → 900명 → 2,700명 (3단계만에 3,000명+)
- 초대받은 사람의 첫 답변 완료율 추적 → 초대장 추가 지급 조절

**Phase 4 — 오픈 (대기열 소진)**
- 초대제 유지하되 대기열 등록자에게 순차 오픈
- 오픈 후에도 "초대하면 친구가 답변 시 초대장 +1" 메커니즘 유지

### 11-3. 심리적 후킹 전략 (유저를 놓지 않는 설계)

**A. 진입 시 — "나도 답하고 싶다" 만들기**

| 트리거 | 심리 원리 | 구현 |
|---|---|---|
| 질문 자체가 강력 | 호기심 갭 (Curiosity Gap) | 질문을 읽으면 답을 이미 알고 있는 느낌 → 말하고 싶어짐 |
| "You already know your answer" | 자아 인식 자극 | 답을 알고 있는데 안 말하고 있다는 걸 자각시킴 |
| "47명이 지금 녹화 중" | 사회적 증거 (Social Proof) | 다른 사람들이 하고 있으니 나도 해야 할 것 같음 |
| "12시간 후 사라짐" | 손실 회피 (Loss Aversion) | 안 하면 기회를 잃는다는 느낌 |
| 초대장 3장 | 희소성 + 특별함 | 내가 선택된 느낌, 초대해준 사람에 대한 사회적 의무감 |

**B. 첫 답변 시 — "생각보다 괜찮네" 만들기**

| 트리거 | 심리 원리 | 구현 |
|---|---|---|
| 질문이 있으니 뭘 말할지 안 고민 | 인지 부하 감소 | "Just answer" — 생각할 게 없음 |
| 편집/필터 없음 | 완벽주의 제거 | 잘 찍을 수 없으니까 잘 찍으려는 부담 자체가 사라짐 |
| 1분 제한 | 낮은 시간 투자 | "1분만 솔직하면 돼" — 진입 비용 극소 |
| 카메라 자유 | 자기 노출 부담 감소 | 얼굴 안 비춰도 됨 → 심리적 안전망 |
| 답변 후 피드 열림 | 즉각적 보상 | "답하면 다른 사람들 답변 볼 수 있어" → 호기심이 보상 |

**C. 답변 후 — "또 하고 싶다" 만들기**

| 트리거 | 심리 원리 | 구현 |
|---|---|---|
| 공명 알림 | 간헐적 보상 (Variable Reward) | 언제 올지 모르는 "나도 그래요" → 슬롯머신 효과 |
| 프리셋 메시지 | 감정적 연결 | "좋아요" 가 아니라 "용기가 돼요" → 감정이 전달됨 |
| 서로 공명 | 상호성 원리 (Reciprocity) | 내가 공명한 사람이 나한테도 공명 → 연결감 |
| 다른 사람 답변 보기 | 사회적 비교 + 공감 | "나만 이런 생각 하는 게 아니었네" → 소속감 |
| 나도 답하기 유도 | 모방 본능 (Mimicry) | 남의 솔직함이 나의 솔직함을 촉발 |

**D. 장기 — "떠날 수 없다" 만들기**

| 트리거 | 심리 원리 | 구현 |
|---|---|---|
| 개인화 질문 | 관여 상승 효과 (Escalation) | AI가 내 이야기를 알고 있음 → 떠나면 이 관계가 끊김 |
| 아카이브 | 매몰 비용 (Sunk Cost) | 답변이 쌓일수록 떠나기 어려움 — 내 기록이 여기 있으니까 |
| 1년 전 오늘 | 향수 + 자기 성찰 | 과거의 나를 다시 들을 수 있다는 기대감 |
| 사람 발견 | 네트워크 효과 | 답변할수록 더 재밌는 사람을 만남 → 가치가 계속 올라감 |
| 서로 공명 → 귓속말 | 관계 투자 | 여기서만 만난 사람이 생기면 떠날 수 없음 |

**E. 이탈 방지 — "안 해도 되지만 계속 생각남" 만들기**

- 스트릭/압박 알림 **절대 없음** → 죄책감이 아니라 자발적 복귀 유도
- 대신 질문 자체가 궁금해서 앱을 여는 구조
- 정오에 질문 알림 → "오늘 질문 뭐지?" 호기심
- 자정에 개인 질문 알림 → "AI가 나한테 뭐라고 물어봤지?" 호기심
- 답 안 해도 질문은 볼 수 있음 → 보다 보면 답하고 싶어짐
- **"안 해도 되는데 하고 싶은"** 이 중독의 가장 강력한 형태

### 11-4. 앱 내 바이럴 루프

1. **나도 답하기**: 남의 답변 보기 → 나도 같은 질문에 답하고 싶어짐
2. **질문 스크린샷**: 오늘의 질문 자체가 공유 가능한 콘텐츠 ("오늘 Spill 질문 실화냐...")
3. **공명 알림**: 모르는 사람한테 "나도 그래요" 받는 경험이 감정적으로 중독적
4. **서로 공명**: 진짜 연결이 만들어짐 → 입소문
5. **초대장 공유**: "나 초대장 남았는데 줄까?" → SNS에서 자연 바이럴
6. **답변 클립 공유**: 내 답변을 인스타 스토리/트위터에 공유 (30초 프리뷰 + "나머지는 Spill에서")

### 11-5. 리텐션 요약

| 메커니즘 | 효과 | 심리 원리 |
|---|---|---|
| 하루 2개 질문 | 앱 열 이유 2번 | 호기심 |
| 12시간 만료 | 긴박감 (죄책감은 아님) | 손실 회피 |

## 12. AI 연동 스펙

### 12-1. 개인화 질문 생성

**트리거**: 유저가 새 답변을 업로드할 때마다
**동작**: 다음 날 개인화 질문 생성 후 DB에 저장
**폴백**: 답변 없는 날 → 전날 개인화 질문 재사용 or 일반 풀에서 선택

**프롬프트 구조**:
```
System: 너는 친한 친구처럼 편하게 질문을 던지는 역할이야.
존댓말이지만 딱딱하지 않게.
설문지나 심리테스트 느낌 절대 금지.
질문은 1문장, 30자 이내.

User: 아래는 이 사람의 최근 답변들이야.
[답변 1 텍스트 요약]
[답변 2 텍스트 요약]
...

이 사람에게 오늘 던질 질문 1개를 만들어줘.
이전 답변의 내용을 자연스럽게 이어받아서 꼬리를 무는 질문이면 좋아.
친구가 자연스럽게 물어볼 법한 느낌.
```

### 12-2. 음성-텍스트 변환 (STT)

- 업로드된 모든 영상을 STT로 텍스트 변환
- 변환 텍스트 DB 저장 (개인화 질문 생성에 활용)
- 텍스트는 **유저에게 절대 노출 안 함** — 내부 AI 용도만
- STT 벤더: 미정 (Whisper / Google STT / Clova)

### 12-3. 공통 질문 생성

- 미정: 운영팀 직접 작성 vs AI 생성 + 운영팀 승인
- 품질 기준: 읽는 순간 "나 이거 답하고 싶다"가 나와야 함
- 회피: 뻔한 것, 지루한 것, 설문 느낌, 너무 무거운 것, 너무 가벼운 것

| D | **Nothing but you.** | 너만 있으면 돼 | 미니멀, 편안함, 필터 없음 암시 |
| E | **Just spill it.** | 그냥 털어놔 | 브랜드명과 직결, 캐주얼 |
| F | **Unscripted.** | 대본 없음 | 한 단어, 강렬, 편집/필터 없음 암시 |

현재 랜딩에 적용된 슬로건: **A. Real you. Real me.**

## 18. 질문 뱅크 (공통 질문 풀)

### 운영 원칙
- 초기에는 운영팀이 직접 작성, 1주일치(7개) 미리 준비
- 매일 정오에 1개 발사
- 유저 반응 데이터 수집 (답변율, 공명수, "나도 답하기" 수)
- 3개월치 데이터 쌓이면 → AI 생성 + 운영팀 큐레이션으로 전환
- **패턴을 골고루 섞는 것이 핵심** — 같은 패턴이 연속으로 나오면 지루해짐

### 질문 믹스 비율 (1주일 기준)
- 패턴1 "아 맞아 나도" × 2개
